Roles - DB2

As roles funcionam para simplificar a administração e o gerenciamento de privilégios à usuários, oferecendo uma capacidade equivalente a grupos, porém sem as mesmas restrições, ou seja, a role é um objeto no banco que agrupo um ou mais privilégios de usu´rios, podendo ser atribuída a N usuários, bastando inseri-los em um grupo em que a role está presente. As roles oferecem várias vantagens que facilitam o gerenciamento de privilégios em um sistema de banco de dados, onde:

  • Os administradores podem controlar o acesso aos seus bancos de uma maneira que espelha a estrutura de suas organizações.
  • O fácil manipulamento para conceder ou negar permissões.
  • Simplificação de grant's, ois ao invés de conceder a mesma permissão a N usuários, basta inseri-los numa mesma role que contêm tal permissão.
  • É possível delegar o gerenciamento de uma role para outras.
  • Todos os privilégios podem ser armazenados em uma role.

OBS: É possível delegar o gerenciamento da role para outro usuários, podendo o mesmo controlar a associação da funçlção.

Porém, existem uma restrição na criação de role, onde a mesma não pode possuir objetos.

O DBA tem autoridade para criar, decartar, conceder, revogar e comentar uma role, utilizando do GRANT para conceder a participação em uma role à uma ID ou REVOKE para negar.

Após a criação da role, é possível usar do GRANT para atribuir autoridades/privilégios à role., como por exemplo:

  1. Criação da role
    2.                 CREATE ROLE name
  2. Concedimentos a role
    3.                 GRANT privilegio ON TABLE objeto TO ROLE name
  3. Atribuir usuários à role
    4.                 GRANT ROLE name TO USER usuário
  4. Retirar usuários à role
    5.                 REVOKE ROLE name TO USER usuário

De forma a facilitar ainda mais a administração de permissões, é possível também conceder a uma role, todos os privilégios de outra role, o que é chamado de hierarquia das roles. Imagens que na role X é concedido osprivilégios A e B, já na role Y são concedidos os privilégiosda role X e os privilégios C e D. Realizado pelo comando:

            GRANT ROLE role 1 TO ROLE role 2
