WLM - DB2

No BD2 há uma forma de criar uma monitoração que executa uma ação através de parâmetros definidos pelo DBA. Este é um procedimento com o uso de várias funções do DB2. sendo eles:

  • Threshould

    • Usados para monitorar o uso de armazenamento do Banco de dados. Podendo definir threshoulds de aviso/alerta. Por exemplo, se um object storage exceder os níveis ou limimites definidos, um sinalizador será exibido.

    Quando uma database é criada, automaticamente são incrementadas nele, as default threshoulds para seus objetos. Sendo possível sobrescreve-los, fornecendo valores específicos para qualque um dos objetos. Neste contexto, são monitorados 3 tipos de limites, sendo eles:

    1. Space Usage
      2.

      Ou uso de espaço, mede a porcentagem de espaço de armazenamento disponível, usado por um objeto (monitorado por tablespaces), representado como uma porcentagem do armazenamento total.

    2. Data Skew
      3.

      Ou Distorção de dados, mede a distribuição de ddos, medindo o desvio de um objeto do nível médio de dados (monitorado por tables/databases partitions).

    3. Cluster Ratio
      4.

      Ou taxa e cluster, mede o grau em que as linhas em uma tabela são organizadas na mesma ordem especificada por um determinado índice, onde uma alta porporção de cluster, indica que as linhas de dados são armazenadas na mesma sequência física. Já se há uma baixa proporção, é indicado que são armazenados em uma sequência física diferente.

  • Event Monitor

    • Retornam informações de eventos especificados pelo DBA, ou seja, funciona como uma captura de determinadas informações desejadas. Estes eventos são gerados por instruções de DML. Além disso, 3 passos básicos são seguidos (I - Criação, II - Habilitação, III - Acesso).

  • Service Class

    • Gerenciamento das funções do workload e do threshould.

  1. Criação da tablespace para armazenar os dados do event monitor (Bufferpool + Tablespace)

  2. Criação do event monitor

  3. Habilitação do event monitor
